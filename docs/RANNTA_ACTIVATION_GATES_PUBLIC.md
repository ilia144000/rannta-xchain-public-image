# Public Activation Gates

The RANNTA Network activation model uses explicit gates.

Public summary:

- A1 Operator approval
- A2 Private dedicated install
- A3 Private runtime health and persistence proof
- A4 Exchange internal service mount
- A5 Read-only market API readiness
- A6 Settlement and fund movement go/no-go
- A7 Public activation decision

The public image intentionally excludes private commands, private endpoints, private tokens, and production operator details.
